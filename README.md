# 🚀 Spring AI Copilot Prompts

Conjunto de prompts estruturados para uso com IA, adaptados para o ecossistema **Java + Spring Boot**.

Este repositório reúne diferentes modos de interação com IA para desenvolvimento de software, focando em **produtividade, qualidade de código e aprendizado guiado**.

---

## 💡 Ideia do projeto

A proposta é utilizar IA como um verdadeiro **copiloto de desenvolvimento**, com comportamentos bem definidos dependendo do contexto.

Ao invés de usar prompts genéricos, aqui você encontra modelos organizados que orientam a IA a agir de forma mais precisa, como:

* explicar conceitos
* planejar implementações
* gerar código com qualidade
* auxiliar no aprendizado

Todos os prompts foram adaptados para o stack:

> **Java 21 + Spring Boot**

---

## 🧠 Modos disponíveis

### 🔹 ASK

Modo de dúvidas rápidas e diagnóstico.

* Explica erros e códigos
* Sugere abordagens
* Não implementa diretamente

---

### 🔹 PLAN

Modo de planejamento técnico.

* Gera planos estruturados
* Define arquitetura e passos
* Levanta riscos e validações antes de codar

---

### 🔹 AGENT CODE

Modo de execução.

* Implementa funcionalidades completas
* Gera código pronto para uso
* Inclui testes e validações

---

### 🔹 STUDY

Modo de aprendizado.

* Explica conceitos em profundidade
* Usa analogias e exemplos
* Foco em entendimento real, não só solução

Criar um **playbook reutilizável de prompts** para:

* acelerar desenvolvimento backend

* servir como apoio em estudos

## 📚 Contexto

Este repositório foi criado como parte de uma atividade de um bootcamp, com o objetivo de adaptar prompts genéricos para uma stack específica.

Apesar disso, o conteúdo foi estruturado para ser útil em **projetos reais e no dia a dia de desenvolvimento**.

---

## 🚀 Como usar

1. Escolha o modo (ASK, PLAN, AGENT ou STUDY)
2. Copie o prompt correspondente
3. Cole na IA junto com sua dúvida ou tarefa
4. Ajuste conforme necessário para seu contexto

---

## 📌 Observações

* Os prompts podem ser evoluídos conforme a necessidade
* Você pode adaptar para outras stacks (Quarkus, Micronaut, etc.)
* O foco aqui é **qualidade de interação com IA**, não apenas código
